# 견적 폼 — 간격·크기·정렬 디자인 스펙

목적: 가독성 ↑, 공간 효율 ↑, 일관성 확보. 아래 수치를 고정값으로 쓰고, 새 필드를 추가할 때도 이 표에서만 골라 쓸 것 (임의의 px 금지).

---

## 1. 레이아웃 그리드

- **작성 영역(센터 컬럼) 최대 폭**: `860px`, 가운데 정렬. 이 이상 넓히지 않는다 — 한 줄에 너무 많은 텍스트가 들어오면 가독성이 떨어짐.
- **카드(섹션) 사이 세로 간격**: `22px` (컬럼 전체를 `flex-direction:column; gap:22px`로 관리 — 카드마다 다른 margin 주지 말 것).
- **카드 내부 패딩**: `28px 30px` (상하 28 / 좌우 30) 고정.
- **카드 모서리 radius**: `10px`. 필드 컨트롤(입력창 등) radius는 `6px`로 카드보다 작게 — 중첩된 둥근 모서리는 하나로 통일하지 않는다(카드는 크게, 내부 요소는 작게).

## 2. 필드 그리드 (섹션 내부)

- 기본 **2열 그리드**: `grid-template-columns: 1fr 1fr; gap: 20px 22px` (세로 20 / 가로 22).
- **1열 전체폭(span2)로 배치해야 하는 위젯 타입**: 멀티셀렉트(칩), 텍스트에어리어, 치수(가로×세로), 예/아니오 세그먼트, 주소.
  → 이유: 칩은 줄바꿈이 필요하고, 세그먼트/치수는 좁은 컬럼에서 눌리면 가독성이 깨짐.
- **1열(절반폭)에 둬도 되는 타입**: select, 단문 text, 숫자+단위, date.
- 모바일(≤640px)에서는 그리드를 강제로 1열로 전환.

## 3. 필드 컴포넌트 내부 스펙

- 필드 하나 = `label → (help, 선택) → widget → error(숨김)` 세로 스택, 각 요소 사이 간격 `7px` 고정.
- **라벨**: 13.5px / 600 weight. 필수 표시는 배지가 아니라 라벨 옆 빨간 별표(`*`) 하나로 — 배지를 남발하면 화면 전체가 "경고"처럼 보임.
- **도움말(help)**: 라벨 바로 아래, 12.5px, 여백은 위쪽만 `-2px`로 살짝 붙여 라벨과 한 그룹처럼 보이게.
- **모름/추천요청 pill**: 라벨과 같은 줄, 우측 정렬(`justify-content:space-between`). 필드마다 각각 배치하지 말고 라벨 행에 통합.
- **에러 메시지**: 위젯 아래, 11.5px, 평소엔 숨김(`display:none`) → 유효성 실패 시에만 노출. 항상 자리를 차지하게 만들지 말 것(레이아웃 흔들림 방지는 min-height로 대응하거나, 실패 상태에서만 카드에 여유를 둔다).

## 4. 컨트롤(입력 요소) 크기

| 컨트롤 | 높이/크기 | 비고 |
|---|---|---|
| text / select / number / date / tel | 44px | 폰트 14px, 좌우 패딩 14px |
| textarea | 최소 높이 88px | 세로 리사이즈 허용, 패딩 12px 14px |
| 멀티셀렉트 칩 | 상하 9px 좌우 15px | pill 모양, 칩 사이 gap 8px |
| 예/아니오 세그먼트 버튼 | 상하 10px 좌우 19px | 2버튼 붙여서 하나의 컨트롤처럼 |
| 치수(가로×세로) 입력 | 44px, 가운데 정렬 텍스트 | 사이에 `×` 구분자 + 우측에 단위(mm) 고정 텍스트 |
| 숫자+단위 (개/㎡/mm) | 44px | 입력 옆에 단위를 별도 텍스트로 붙임(토글 아님) |
| 버튼(분야 제거, 임시저장 등) | 44px | 텍스트 버튼은 패딩 7px 10px |

→ **모든 입력형 컨트롤은 44px로 통일.** 옆에 나란히 두는 버튼(주소검색 등)도 반드시 44px로 맞춰 세로 정렬을 깨지 않는다.

## 5. 섹션(카드) 헤더 규칙

- 제목 18px bold + 완료 분수 `(n/m 완료)` 12.5px를 같은 줄, `align-items:baseline`으로 텍스트 베이스라인 정렬 (세로 중앙정렬 아님 — 크기 다른 텍스트를 baseline으로 맞춰야 자연스러움).
- 헤더와 우측 액션(분야 제거 등)은 `justify-content:space-between`으로 카드 폭 양 끝에 배치.
- 한 카드 안에 여러 정보 블록이 있을 때(예: 공통정보의 "기본 정보" / "일정 및 연락처")는 블록 사이에 `margin-top:26px; padding-top:26px; border-top:1px solid`로 구분 — 카드를 여러 개로 쪼개지 않는다.

## 6. 타이포그래피 스케일 (이 6단계만 사용)

1. 페이지 타이틀: 24–27px / 700
2. 섹션 제목: 17–18px / 700
3. 필드 라벨 / 카드 소제목: 13.5–14px / 600–700
4. 본문·컨트롤 텍스트: 14px / 400
5. 도움말·보조 텍스트: 12–12.5px / 400–500
6. 뱃지·캡션·에러: 11–11.5px / 600–700

## 7. 정렬 원칙

- 모든 가로 배치(칩, 버튼, 아이콘+텍스트 등)는 **flex + gap**으로. margin으로 간격을 만들지 않는다 (요소 추가/삭제 시 깨짐).
- 라벨-위젯-에러는 항상 좌측 정렬. 우측 정렬은 "모름 pill", "분야 제거", 섹션 액션 버튼 등 보조 액션에만 사용.
- 체크박스·라디오류는 텍스트와 세로 중앙 정렬(`align-items:center`), 텍스트 블록(라벨+설명 2줄)과는 baseline이 아니라 center 정렬.

## 8. 자주 하는 실수 (체크리스트)

- [ ] 입력창 높이가 필드마다 제각각이다 → 44px로 통일했는가
- [ ] 어떤 필드는 카드 패딩이 다르다 → 28px 30px로 통일했는가
- [ ] 라벨 크기가 화면마다 다르다 → 13.5–14px 스케일을 따랐는가
- [ ] 필수 표시를 빨간 배지로 여러 개 남발했다 → 별표(`*`) 하나로 정리했는가
- [ ] 칩/세그먼트를 절반 폭 컬럼에 욱여넣었다 → span2 규칙을 지켰는가
- [ ] 카드 사이 간격이 요소마다 다른 margin으로 되어있다 → 부모에 `gap:22px`로 통일했는가
- [ ] 버튼과 입력창을 나란히 뒀는데 높이가 다르다 → 둘 다 44px인가
