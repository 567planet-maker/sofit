## 5. 접근 제어 / RLS — 타인 데이터 차단

> 시나리오: 권한 없는 URL을 직접 입력했을 때 막히는지 확인

- [ ] 고객 A로 로그인 → 고객 B 요청 URL 직접 입력 (`/customer/requests/[B id]`) → 404 확인
- [ ] 공장으로 로그인 → 매칭 안 된 요청 상세 URL 직접 입력 → 404 확인

- [ ] 고객 계정으로 → `/admin` 직접 입력 → `/customer/me`로 리디렉션 확인
- [ ] 공장 계정으로 → `/customer/me` 직접 입력 → `/factory`로 리디렉션 확인
- [ ] 로그아웃 후 이전 보호 URL 재접근 → 로그인으로 차단 확인

## 7. 채팅 실시간

> 준비: 같은 채팅방을 탭 2개로 열기 (좌측 목록에서 동일 방 선택)

- [ ] 탭 A(고객)에서 메시지 전송 → 탭 B(관리자)에 즉시 수신 (새로고침 없이)
- [ ] 탭 B에서 메시지 전송 → 탭 A에 즉시 수신
- [ ] 메시지 도착 시 자동 스크롤 확인
- [ ] 탭 B에서 읽음 처리 → `read_at` 갱신 확인 (Supabase)
- [ ] 파일(이미지) 첨부 전송 → 상대 탭에 이미지 노출

## 10. 모바일 반응형

> Chrome → F12 → 기기 에뮬레이터 (iPhone 14 Pro / Galaxy S23 권장)

- [ ] `/` 랜딩 — 모바일 레이아웃 깨짐 없음, 헤더 메뉴 정상
- [ ] `/customer/request` 폼 — 스텝별 입력 가능, 버튼 터치 크기
- [ ] **마이페이지/공장관리** — 좌측 패널 숨김 + 상단 가로 스크롤 네비로 대체되는지
- [ ] **채팅** — 모바일에서 목록/대화 표시, 입력창이 키보드에 가려지지 않는지
- [ ] `/customer/requests/[id]/quotes` — 견적서 카드 가독성
