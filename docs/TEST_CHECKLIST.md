# SOFIT W8 — 통합 테스트 체크리스트

> 테스트 환경: 운영 URL (Vercel 배포본)  
> 준비물: 카카오 계정 3개 (고객용·공장용·관리자용)  
> 권장 브라우저: Chrome (시크릿 창으로 계정 분리)

---

## 사전 준비

- [ ] 관리자 계정 로그인 확인 (`/admin` 접근 가능)
- [ ] Supabase 대시보드 열어두기 (테이블 확인용)
- [ ] Chrome 시크릿 창 3개 준비 (고객 / 공장 / 관리자)

---

## 1. 고객 E2E

> 시나리오: 처음 방문한 고객이 견적을 요청하고 계약까지 확정하는 전 과정

### 1-1. 회원가입 + 온보딩
- [ ] `/` 접속 → Hero 섹션 CTA 버튼 노출 확인
- [ ] "견적 요청하기" 클릭 → `/login` 이동
- [ ] 카카오 로그인 버튼 클릭 → 카카오 로그인 완료
- [ ] `/onboarding` 로 이동 확인
- [ ] 약관 동의 체크 전에는 역할 선택 비활성화 확인
- [ ] 약관 동의 체크 → "고객" 선택 → 완료
- [ ] `/customer` 대시보드로 이동 확인

### 1-2. 견적 요청 폼 제출
- [ ] `/customer/request` 접속
- [ ] **Step 1** — 업체명·현장명·주소·담당자·연락처·방문가능시간 입력 후 다음
- [ ] **Step 2** — 쇼파 형태·수량 등 선택 후 다음 (선택사항이므로 스킵 가능)
- [ ] **Step 3** — 규격 입력 또는 도면 파일 업로드 (pdf 1개)
- [ ] **Step 4** — 현장 사진 업로드 (jpg 1개)
- [ ] **Step 5** — 납품 희망일 선택 → "제출하기"
- [ ] 제출 완료 페이지 이동 확인
- [ ] Supabase `quote_requests` 테이블에 레코드 생성 확인 (status: submitted)
- [ ] Supabase `chat_rooms` 테이블에 customer_sofit 방 생성 확인

### 1-3. 상태 확인 + 채팅
- [ ] `/customer/requests` — 제출한 요청 카드 노출 확인
- [ ] 요청 카드 클릭 → 상세 페이지 진행 이력(status_logs) 확인
- [ ] "소핏 담당자와 채팅" 버튼 클릭 → 채팅방 이동
- [ ] 메시지 입력 + 전송 확인

### 1-4. 견적서 수신 + 수락
> (공장 E2E 완료 후 진행)
- [ ] 알림 벨에 새 알림 뱃지 확인
- [ ] `/customer/requests/[id]/quotes` 접속
- [ ] 공장별 견적서 카드 노출 확인 (총액·납기 표시)
- [ ] "이 견적서로 계약하기" 클릭 → 2단계 확인 팝업 노출
- [ ] "네, 계약 확정" 클릭
- [ ] 견적서 상태 "계약 확정" 배지로 변경 확인
- [ ] `/customer/requests` — 해당 요청 상태가 `contracted`로 변경 확인
- [ ] 공장 알림 수신 확인 (공장 창에서)

## 2. 공장 E2E

> 시나리오: 공장이 가입하고 승인받은 뒤 견적서를 제출하는 전 과정

### 2-1. 공장 가입 + 심사 대기
- [ ] 새 시크릿 창 → `/login` → 카카오 로그인 (공장용 계정)
- [ ] `/onboarding` → "공장" 선택
- [ ] `/factory/onboarding` — 회사명·위치·소개 입력 후 제출
- [ ] `/factory/pending` 심사 대기 페이지 노출 확인
- [ ] Supabase `factories` 테이블 status = 'pending' 확인

### 2-2. 관리자 승인
- [ ] 관리자 창 → `/admin/factories` → 승인 대기 탭
- [ ] 해당 공장 클릭 → "승인" 버튼 클릭
- [ ] 공장 창 새로고침 → `/factory` 대시보드 진입 확인

### 2-3. 포트폴리오 등록
- [ ] `/factory/portfolios` → "작업 사례 추가" 클릭
- [ ] 이미지 업로드 + 설명 입력 → 저장
- [ ] `/portfolios` (공개 페이지) 에서 해당 포트폴리오 노출 확인

### 2-4. 신규 요청 참여 + 견적서 제출
- [ ] `/factory/requests` → "신규 요청" 섹션에 고객 요청 노출 확인
- [ ] 요청 클릭 → 고객 폼 내용 열람 가능 확인
- [ ] "매칭 참여하기" 클릭
- [ ] 페이지 새로고침 → "내 참여 요청" 섹션으로 이동 확인
- [ ] 견적서 작성 폼 노출 확인 (matches.status = confirmed)
- [ ] 항목별 금액 입력 → 총액 자동 계산 확인
- [ ] 납기일 입력 → "임시저장" 클릭 → 새로고침 후 값 유지 확인
- [ ] "고객에게 제출하기" 클릭
- [ ] 고객 창 → 알림 벨 뱃지 증가 확인
- [ ] `quote_requests` 상태 → `quote_arrived` 확인

### 2-5. 수정 견적서 제출 (선택)
- [ ] 동일 매칭에서 금액 수정 후 다시 "고객에게 제출하기"
- [ ] 고객 견적서 화면에서 "수정 견적서 v2" 배지 + diff 테이블 노출 확인

## 3. 관리자 E2E

> 시나리오: 관리자가 요청을 검수하고 공장을 배포·관리하는 전 과정

### 3-1. 새 요청 수신
- [ ] `/admin` 대시보드 → 미처리 건수 뱃지 확인
- [ ] `/admin/requests` → submitted 행 하이라이트 확인
- [ ] 요청 클릭 → 고객 폼 전체·사진·도면 열람 확인
- [ ] 상태 변경 드롭다운 → "검토중" 선택 → 저장
- [ ] 고객 창 진행 이력에 "검토중" 추가 확인

### 3-2. 공장 배포
- [ ] "공장 매칭 관리" 버튼 → `/admin/requests/[id]/match`
- [ ] 활성 공장 목록 노출 확인
- [ ] "배포" 클릭 → 공장 창 알림 확인
- [ ] 매칭 이력에 배포된 공장 노출 확인

### 3-3. 채팅 모니터링
- [ ] `/admin/chats` → 채팅방 목록 노출 확인
- [ ] customer_sofit 방 선택 → 메시지 전송
- [ ] 고객 창 → 소핏 메시지 수신 확인
- [ ] customer_factory 방 → 읽기 전용 (입력창 없음) 확인

## 4. RLS — 타인 데이터 접근 차단

> 시나리오: 권한 없는 URL을 직접 입력했을 때 막히는지 확인

- [ ] 고객 A로 로그인 → 고객 B 요청의 URL 직접 입력 (`/customer/requests/[B의 요청 id]`) → 404 확인
- [ ] 공장으로 로그인 → 매칭 안 된 요청 상세 URL 직접 입력 → 404 확인
- [ ] 비로그인 상태 → `/customer/requests` 직접 입력 → `/login` 리디렉션 확인
- [ ] 비로그인 상태 → `/admin` 직접 입력 → `/login` 리디렉션 확인
- [ ] 고객 계정으로 → `/admin` 직접 입력 → 리디렉션 확인

## 5. 파일 업로드 제한

- [ ] 견적 요청 폼 Step 3 — 11MB 이상 PDF 업로드 시도 → 에러 메시지 노출 확인
- [ ] 견적 요청 폼 Step 4 — `.exe` 등 허용되지 않는 확장자 업로드 시도 → 차단 확인
- [ ] 견적 요청 폼 Step 4 — 11MB 이상 이미지 업로드 시도 → 에러 메시지 노출 확인
- [ ] 정상 파일(5MB jpg) 업로드 → 미리보기 썸네일 노출 확인
- [ ] 업로드된 파일 × 버튼 → 삭제 확인

---

## 6. 채팅 실시간

> 준비: 같은 채팅방 URL을 탭 2개로 열기

- [ ] 탭 A(고객)에서 메시지 전송 → 탭 B(관리자)에 즉시 수신 확인 (새로고침 없이)
- [ ] 탭 B에서 메시지 전송 → 탭 A에 즉시 수신 확인
- [ ] 메시지 도착 시 자동 스크롤 확인
- [ ] 탭 B에서 메시지 읽음 처리 → `read_at` 갱신 확인 (Supabase 테이블)
- [ ] 파일(이미지) 첨부 전송 → 상대 탭에 이미지 노출 확인

---

## 7. 알림 실시간

- [ ] 탭 A(고객)에서 견적 요청 제출 → 탭 B(관리자) 벨 아이콘 뱃지 즉시 증가 확인
- [ ] 탭 B(관리자)에서 상태 변경 → 탭 A(고객) 벨 아이콘 뱃지 즉시 증가 확인
- [ ] 벨 아이콘 클릭 → 드롭다운에 최근 알림 목록 노출 확인
- [ ] 알림 클릭 → 해당 페이지 이동 + 알림 읽음 처리 확인
- [ ] `/notifications` — 전체 알림 목록 페이지 확인
