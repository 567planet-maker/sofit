## 1. 제품 개요

### 1.1 한 줄 정의
**소핏(SOFIT)** 은 상업·주거 공간의 가구·인테리어 시공을 위한 **B2B 견적 매칭 플랫폼**이다. 고객이 표준화된 폼·사진·도면으로 견적을 요청하면, 검증된 제작 공장과 매칭하여 견적서를 비교·계약하고, 시공 전 과정을 한곳에서 관리한다.

### 1.2 해결하는 문제
| 주체 | 페인 포인트 | 소핏의 해결 |
|------|------------|-------------|
| **고객**(카페·식당·사무실 등) | 견적 비교 번거로움, 가격 불투명, 공장 신뢰도 판단 불가 | 표준 폼 1회 작성 → 복수 공장 견적서 비교, 검증된 공장만 노출 |
| **제작 공장**(지역 소규모) | 영업·홍보·견적 작성 부담, 안정적 수주 채널 부재 | 포트폴리오 노출 + 매칭된 요청만 선별 수신, 견적서 직접 작성 |
| **소핏** | — | 양측 매개 + 신뢰 인프라 제공 → 수수료 기반 수익 |

### 1.3 비즈니스 모델
- 거래 성사 시 **수수료 7~10%** (현재 MVP는 거래 검증 목적 무료 운영)
- 매칭 전 소통은 소핏 경유 → 매칭 확정 후 고객↔공장 직접 채팅(소핏 모니터링)
- 공장이 직접 견적서 작성·전달 (소핏 검수 단계 없음, 운영 부담 최소화)

### 1.4 타깃 시장
- **초기 지역**: 부산·경남
- **고객**: 신규 창업 카페·식당 등 상업 공간 (프로젝트당 100~2,000만원대)
- **공급자**: 부산·경남 지역 3~6인 규모 가구·인테리어 제작 공장

## 3. 사용자별 구현 기능 명세

### 3.1 인증 · 계정 (공통)
- **소셜/이메일 로그인**: 카카오 OAuth(자체 구현) + 이메일 회원가입(이메일 인증). 네이버는 연동 스캐폴드 보유.
- **역할 온보딩**: 가입 후 고객/공장 선택 → 역할별 라우팅. 공장은 추가 심사 단계.
- **프로필**: 이름·프로필 사진 설정(채팅·마이페이지 반영).
- **계정 관리**: 계정 정보 수정, **회원 탈퇴**(서버측 완전 삭제).
- **로그인 복귀**: 보호 페이지 진입 시 로그인 후 원래 위치로 복귀.
- **이용약관·개인정보처리방침**: 동의 + 게시 페이지(`/terms`, `/privacy`).

### 3.2 고객(Customer)
| 기능 | 설명 |
|------|------|
| 견적 요청 폼 | 멀티스텝 위저드(현장→제품→규격·도면→자재·사진→일정). 입력 임시저장(localStorage). |
| 파일 업로드 | 매장 사진(JPG/PNG/WEBP), 도면(PDF/DWG/DXF). 용량·확장자 제한. |
| 내 견적 현황 | 요청 목록(상태 탭 필터) + 진행 타임라인. |
| 견적 상세 | 요청 내용·첨부·상태 이력·받은 견적서 요약. |
| 견적서 비교·수락 | 복수 공장 견적서를 **버전별 카드**로 비교 → 원하는 공장 **계약 수락**. |
| 위시리스트 | 관심 공장/포트폴리오 저장. |
| 소핏 채팅 | 매칭 전 고객↔소핏 실시간 문의. 매칭 후 고객↔공장 직접 채팅. |
| 공개 포트폴리오 | 로그인 없이 공장 작업 사례 갤러리 열람. |

### 3.3 공장(Factory)
| 기능 | 설명 |
|------|------|
| 가입·심사 | 사업자등록증 업로드 → `승인 대기` → 관리자 승인 시 `활성`. 미승인 시 대기 화면. |
| 포트폴리오 관리 | 작업 사례 등록·수정·삭제(사진·설명·카테고리·완성연월). 전체 공개. |
| 매칭 요청 확인 | 소핏이 배포한 요청 목록·상세(고객 정보 일부 + 폼 + 사진). |
| 수락/거절(자율 참여) | 요청 검토 후 참여 결정. 수락 시 견적서 작성 가능. |
| 수정 견적서 작성 | 자재비/제작비/운송비/설치비/철거비/추가비/마진 + **버전 관리**. 고객에게 직접 전달. |
| 진행 프로젝트 관리 | 진행 중 프로젝트 목록 + **진행 사진 업로드**(포트폴리오·분쟁 증거 활용). |
| 고객 채팅 | 매칭 확정 후 고객과 직접 실시간 협의. |

### 3.4 관리자(Admin / 소핏 백오피스)
> AI 없이 사람이 전 과정을 운영하므로 백오피스가 핵심.

| 기능 | 설명 |
|------|------|
| 전체 견적 요청 관리 | 상태별 필터, 요청 상세, **상태 변경 + 관리자 메모**. |
| 공장 배포(매칭) | 요청을 적합 공장들에게 수동 배포(복수 동시 가능). |
| 최종 매칭 확정 | 고객 선택 확인 → 매칭 확정 → 직접 채팅방 생성, 미선택 공장 알림. |
| 공장 관리 | 가입 심사(사업자등록증·포트폴리오) 승인/반려, 활성/정지. |
| 사용자 관리 | 고객·공장 계정 목록·상태 관리. |
| 채팅 모니터링 | 고객↔소핏 응대 + 고객↔공장 채팅 열람(분쟁 중재). |
| 알림 관리 | 신규 요청 등 관리자 알림. |

## 4. 핵심 시스템 (공통 인프라)

### 4.1 견적 워크플로우 (상태 8단계 + 임시저장)
```
[임시저장] → 접수됨 → 검토중 → 매칭중 → 견적서 도착 →
소통중 → 계약완료 → 시공중 → 완료
```
- DB `status` 컬럼 관리 + 상태별 배지(색상) + **변경 이력(status_logs, audit)**.
- 견적서는 **버전 관리**(공장이 수정본 누적, 고객은 최신본 비교).

### 4.2 실시간 채팅
- 채널 3종: 고객↔소핏(`customer_sofit`), 공장↔소핏(`factory_sofit`), 고객↔공장(`customer_factory`).
- Supabase Realtime 기반, **파일 첨부**, 읽음 처리, 참여자 프로필 표시.

### 4.3 파일·스토리지 (Supabase Storage)
- 매장 사진 / 도면 / 사업자등록증 / 포트폴리오 / 진행 사진 / 채팅 첨부.
- **권한 분리**: 매장 사진=고객·매칭 공장·관리자, 사업자등록증=관리자 전용, 포트폴리오=공개, 채팅 첨부=참여자 한정.

### 4.4 알림 (웹 내)
- 벨 아이콘 뱃지(미읽음 카운트), 클릭 시 해당 페이지 이동.
- 트리거: 새 요청(관리자), 새 매칭(공장), 견적서 도착(고객), 새 메시지(전원), 상태 변경.

### 4.5 보안 (RLS 기반 다층 방어)
- 전 테이블 **Row Level Security** 적용: 고객은 본인 데이터, 공장은 매칭된 요청만, 관리자는 전체.
- 서버 액션마다 인증·소유권 검증(프론트 가드에 의존하지 않음).
- 서버 전용 시크릿(`service_role`)과 공개 키 분리.

## 6. 진행 중인 핵심 확장 — 인테리어 전 분야 견적

> 현재 제품의 가장 중요한 확장. "쇼파 전용 매칭" → **인테리어 전 분야 매칭 플랫폼**으로 시장을 넓힌다.

### 6.1 확장 내용
- **27개 시공 분야** 체계화(철거·설비·목공·바닥·도배·도장·조명·주방·욕실·간판·청소 등 공정 흐름순 8개 그룹).
- **분야별 동적 견적 폼**: 분야를 고르면 그 분야에 맞는 입력 항목만 노출. 한 요청에 **복수 분야** 선택 가능(예: 바닥+도장+조명).
- **"모름/상담/실측" 안전장치**: 전문 용어를 몰라도 견적 요청 가능(고객 진입장벽 완화).
- **분야별 업체 매칭**: 요청을 분야 단위로 적합 공장에 매칭, 분야별 견적서 비교.

### 6.2 기술 설계 (확장성 핵심)
- 데이터: **부모(공통) + 분야별 항목(items) + JSONB 상세** 하이브리드 구조 → 분야가 늘어도 **DB 스키마 변경 없이** 확장.
- **스키마 레지스트리(FIELD_SCHEMAS)** + **동적 렌더러** → 새 분야 추가 = "스키마 한 덩어리 추가"로 끝. 폼·상세 화면 코드 수정 불필요.

### 6.3 구현 진행 현황
| 단계 | 내용 | 상태 |
|------|------|------|
| 데이터 모델·스키마 | 27개 분야 정의, 8개 분야 입력 스키마 완성 | ✅ 완료 |
| DB 마이그레이션 | items/attachments/revisions 테이블 + RLS | ✅ 완료(적용) |
| 서버 로직 | 임시저장·분야저장·검증·제출 액션 | ✅ 완료 |
| 입력 UI(스튜디오) | 3분할 동적 폼 화면 | 🔄 진행 중 |
| 상세 3화면 동적화 | 고객/공장/관리자 분야별 표시 | ⏳ 예정 |
| 분야별 매칭·견적 | 공장↔분야 매칭, 분야별 견적 비교 | ⏳ 예정 |

## 9. 경쟁력 요약 (사업계획서 포인트)

1. **표준화된 견적 인프라** — 사진·도면·요청을 구조화해 비교 가능한 견적서로 전환. 정보 비대칭 해소.
2. **신뢰 기반 매칭** — 심사 통과 공장만 노출, 매칭 전 신원 비공개로 우회 거래 방지.
3. **확장 가능한 아키텍처** — 분야가 늘어도 스키마 추가만으로 대응. 인테리어 전 분야로 시장 확장이 구조적으로 설계됨.
4. **데이터 축적 → AI 전환 경로** — 수동 운영으로 모은 실거래·견적 데이터가 AI 가견적의 학습 자산.
5. **풀스택 운영 완비** — 고객·공장·관리자 3면 + 채팅·알림·상태관리까지 운영 가능한 형태로 이미 구현.
