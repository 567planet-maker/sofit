## Phase 0 — Core Platform (지금 만드는 것, M+0~M+2)

### ✅ IN (반드시 포함)

| 기능 | 설명 |
|------|------|
| 랜딩페이지 | 서비스 소개, 견적 요청 CTA, 포트폴리오 둘러보기 |
| 소셜 로그인 | 카카오 + 네이버 (Supabase Auth) |
| 역할 분리 | 고객 / 공장 / 관리자 3종 RBAC |
| 이용약관·개인정보처리방침 | 가입 시 동의, 법적 의무 페이지 게시 |
| 견적 요청 폼 | 10개 카테고리, 멀티스텝, React Hook Form + Zod |
| 파일 업로드 | 매장 사진 3~10장, 도면·PDF, Supabase Storage |
| 내 견적 현황 | 고객이 요청 목록·상태·진행 타임라인 확인 |
| 견적서 비교 | 1~3개 공장 견적서 카드 비교 (가격·납기 고객에게만 공개) |
| 공장 가입·심사 | 사업자등록증 업로드, 관리자 승인 플로우 |
| 공장 포트폴리오 | 작업 사례 등록·수정, 전체 공개 (비로그인 열람 가능) |
| 수정 견적서 작성 | 공장이 수락 후 항목별 금액 입력·고객에게 직접 전달 |
| 공장 수락/거절 | 매칭 알림 수신 후 요청 검토 → 수락 또는 거절 |
| 진행 사진 업로드 | 공장이 작업 진행 사진 업로드 |
| 관리자 견적 목록 | 상태별 필터, 전체 요청 관리 |
| 관리자 공장 배포 | 견적 요청을 적합한 공장들에게 수동 배포 |
| 관리자 매칭 확정 | 고객 선택 후 최종 매칭 확정 처리 |
| 관리자 공장 승인 | 사업자등록증 검토 후 승인/반려 |
| 채팅 시스템 (매칭 전) | 고객↔소핏 실시간 채팅 (Supabase Realtime) |
| 채팅 시스템 (매칭 후) | 고객↔공장 직접 실시간 채팅 (소핏은 모니터링) |
| 웹 내 알림 | 벨 아이콘 뱃지, 클릭 시 해당 페이지 이동 |
| 상태 관리 | 견적 상태 8단계, 변경 이력(audit log) |
| RLS 보안 | 역할별 데이터 접근 분리 |

### ❌ OUT (Phase 0에서 제외)

| 기능 | 제외 이유 | 추가 시점 |
|------|-----------|-----------|
| 결제·에스크로 | 토스페이먼츠 연동 복잡도, 먼저 거래 검증 필요 | Phase 0.5 (M+3) |
| 리뷰·평점 | 거래 데이터 없음 | Phase 0.5 (M+3) |
| AS 보증금 자동화 | 결제 시스템 선행 필요 | Phase 0.5 (M+3) |
| 카카오 알림톡 | 웹 알림으로 충분, 솔라피 도입은 나중 | Phase 1+ (M+5+) |
| Make 자동화 | 웹 알림으로 대체 | Phase 0.5 (M+3) |
| AI 가견적 엔진 | 자재 단가 DB 미구축, 실거래 데이터 0건 | Phase 1 (M+4~M+5) |
| 자재 단가 DB | 데이터 수집 중 (천일쇼파 견적서) | Phase 1 (M+4) |
| AI 시각화 | AI 가견적 선행 필요 | Phase 2 (M+6+) |
| 자동 매칭 알고리즘 | 거래·평점 데이터 필요 | Phase 2+ |
| 3D 미리보기 | 장기 과제 | Phase 3+ |
| 앱(PWA) | 웹 안정화 후 | Phase 2+ |
| 수도권 공장 확장 | 부산·경남 안정화 후 | Phase 2+ |

## Phase 0.5 (M+3)

### ✅ 추가

| 기능 | 설명 |
|------|------|
| 토스페이먼츠 결제 | 단계별 결제 30/50/20, 서버 금액 검증 필수 |

| 리뷰·평점 | 완납 후 별점 4항목 + 텍스트 리뷰 |
| 리뷰 인센티브 | 다음 거래 수수료 할인 |
| Make 자동화 | 이메일 알림 트리거 |
| 법무 문서 | 거래 표준 계약서, 환불·취소 정책, AS 보증 정책 |
