## 0. 이 문서의 위치

본 PRD는 **AI를 의도적으로 제외한 1차 개발용 MVP** 문서입니다. AI 가견적·시각화 없이 "사람이 수동으로 운영하는 매칭 플랫폼"을 먼저 완성하고, 그 위에 AI를 점진적으로 얹는 전략입니다.

### 왜 AI를 빼고 시작하는가?

- 자재 단가 DB가 미구축 (AI 견적의 근거가 없음)
- 실제 거래 데이터 0건 (AI 견적 품질 검증 불가)
- 사람(소핏)이 수동으로 견적 쓰면서 데이터를 모으는 게 먼저
- **고객 UX는 동일** — "1~2분 후 가견적 도착"이라고 보여주고, 백엔드에서는 사람이 처리 (Wizard of Oz 방식)

### Phase 진화 로드맵

| 단계 | 기간 | 핵심 |
|---|---|---|
| **Phase 0 (Core, 본 문서)** | M+0 ~ M+2 | 플랫폼 골격, 100% 수동 운영, 결제는 계좌이체 |
| **Phase 0.5** | M+3 | 결제·에스크로 (토스), 리뷰·평점 추가 |
| **Phase 1 (AI 가견적)** | M+4 ~ M+5 | GPT-4o 가견적 엔진, 자재 DB v1 |
| **Phase 2 (AI 시각화)** | M+6 이후 | OpenAI 이미지 합성 |

> ✅ **[확정]** — 즉시 개발에 반영
> 📝 **[검토]** — 개발 중 결정·확인 필요

## 1. 제품 개요

### 1.1 제품 정의

『소핏(SOFIT)』은 상업용 공간(카페·식당·기관)의 쇼파 및 빌트인 가구 발주를 위한 B2B 매칭 플랫폼이다. 고객은 표준 폼과 매장 사진을 통해 견적을 요청하고, 소핏은 적합한 공장과 매칭하여 견적서를 전달한다. 모든 견적·소통·일정·내역은 소핏을 거쳐 통합 관리된다.

> ✅ **[확정]**
> - 플랫폼명: 소핏 (SOFIT)
> - 매칭 성사 전: 고객·공장 소통은 소핏 경유
> - **매칭 성사 후: 고객↔공장 직접 채팅** (소핏은 모니터링 역할)
> - 공장이 직접 수정 견적서를 작성해 고객에게 전달 (소핏 검수 단계 없음)
> - 소핏은 요청 접수 → 공장 배포 → 최종 매칭 확정 역할을 담당
> - 수수료율 7~10% (1차 MVP에서는 수수료 부과 없이 무료 운영, 거래 검증 목적)

### 1.2 핵심 가치 제안 (Core MVP 버전)

- **체계화된 견적서 시스템** — 도면·이미지·요청사항이 통합된 투명한 견적
- **신뢰 기반 매칭** — 선별된 공장만 등록, 소핏이 직접 검수
- ⏸ **[Phase 1+]** 발주 전 AI 시각화 — Phase 2에서 추가

### 1.3 목표

- **고객**: 안심하고 합리적인 가격으로 인테리어를 진행
- **지역공장**: 영업·견적·홍보 부담 없이 안정적 수주 채널 확보
- **소핏**: 양측을 매개하며 신뢰 기반의 수수료 사업 구축

### 1.4 Phase 0 MVP 범위

> ✅ **[확정]**
> - **MVP 핵심 기능**: 랜딩페이지, 로그인, 고객/공장/관리자 화면, 채팅, 포트폴리오, 파일 업로드, 알림, 권한 관리
> - **Phase 0 KPI**: 첫 견적 요청 1건 발생 / 첫 매칭 1건 / 첫 거래 성사 1건 (정성적)
> - **MVP 출시 목표**: 2개월 (M+0 ~ M+2)
> - **실제 거래 목표 (M+2 ~ M+4)**: 1~5건 (수동 운영 검증)

## 2. 타깃 사용자

### 2.1 1차 타깃 — 신규 창업 카페·식당

- 프로젝트당 100~2,000만원 규모
- 디지털 친화적, 모바일 사용 비중 높음
- 페인 포인트: 견적 비교 번거로움, 가격 불투명성

### 2.2 공급자 — 쇼파·빌트인 가구 공장

- 부산·경남 지역 가구 제조 공장
- 3~6인 규모의 소규모 공장
- 페인 포인트: 디지털 홍보·마케팅 어려움

> ✅ **[확정]**
> - 초기 타깃 지역: 부산·경남
> - MVP 협력 공장: 천일쇼파 (부산) — 베타 동의 완료
> - 추가 공장 컨택: M+3 이후

## 3. 핵심 사용자 플로우 (Core MVP)

### 3.1 고객 측 플로우

| 단계 | 행동 | 시스템 동작 |
|---|---|---|
| **1. 진입** | 랜딩 페이지 접속 | 서비스 소개, 견적 요청 CTA, 공장 포트폴리오 둘러보기 |
| **2. 로그인** | 카카오·네이버 소셜 로그인 | Supabase Auth 인증, 이용약관 동의 |
| **3. 정보 입력** | 표준 폼 + 매장 사진 업로드 | 10개 카테고리 폼 수집, Supabase Storage 저장 |
| **4. 요청 제출** | 견적 요청 제출 | 상태: `요청됨` / 관리자에게 웹 알림 |
| **5. 견적 수신** | 수락한 공장들의 수정 견적서 비교 | 상태: `견적서 도착` / 고객에게 알림 |
| **6. 공장 선택** | 마음에 드는 공장 견적서 선택 | 상태: `계약 완료` / 매칭 확정 |
| **7. 직접 소통·진행** | 선택한 공장과 **직접** 채팅으로 협의·진행 | 고객↔공장 직접 채팅방 개설 |
| **8. 완납** | 공장이 진행 사진 업로드, 작업 완료 | 상태: `시공 중` → `완료` |

### 3.2 공장 측 플로우

| 단계 | 행동 | 시스템 동작 |
|---|---|---|
| **1. 가입·심사** | 사업자등록증·포트폴리오 제출 | 상태: `승인 대기` → 관리자 심사 → `활성` |
| **2. 매칭 알림** | 새 매칭 알림 수신 | 웹 알림 (벨 아이콘) |
| **3. 수락/거절** | 요청 내용 검토 후 수락 또는 거절 | 상태: `수락` 시 견적서 작성 가능 |
| **4. 견적서 작성** | 수정 견적서 직접 작성 | 고객에게 직접 전달 (소핏 검수 없음) |
| **5. 매칭 확정** | 고객이 선택 시 최종 매칭 성사 | 고객↔공장 직접 채팅방 개설 |
| **6. 직접 소통·진행** | 고객과 **직접** 채팅으로 일정·상세 협의 | 고객↔공장 직접 채팅 |
| **7. 제작·납품** | 진행 사진 자발적 업로드 | 포트폴리오·AS 분쟁 증거로도 활용 |
| **8. 정산 수령** | 완납 후 계좌이체 수령 | 1차 MVP는 수기 정산 |

> ✅ **[확정]**
> - 매칭 배포는 100% 수동 (소핏 관리자가 공장들에게 요청 배포)
> - 공장이 수락/거절 자율 결정
> - 공장이 직접 견적서 작성 → 고객에게 직접 전달 (소핏 검수 없음)
> - 매칭 성사 후 고객↔공장 **직접 채팅** (소핏은 모니터링)
> - 1차 MVP는 결제 시스템 없이 **계좌이체 수기 처리**

> ⏸ **[Phase 1+]**
> - 결제·에스크로 자동화 (M+3 토스페이먼츠)
> - 리뷰·평점 시스템 (M+3)
> - AI 가견적 (M+4)
> - AI 시각화 (M+6)

## 4. 기능 명세 (Phase 0 Core MVP)

우선순위 표기: **P0**(필수) / **P1**(있으면 좋음)

### 4.1 [P0] 랜딩페이지

서비스 첫인상을 결정하는 페이지. 비로그인 사용자도 접근 가능.

**주요 요구사항:**

- 서비스 소개 (3~5개 섹션)
- 견적 요청 CTA (가입 유도)
- 공장 포트폴리오 둘러보기 링크
- 페인 포인트 → 해결 방식 스토리텔링
- 모바일 반응형

---

### 4.2 [P0] 인증·권한 시스템

**주요 요구사항:**

- **로그인 방식**: 카카오 / 네이버 소셜 로그인 (Supabase Auth)
- **역할(Role) 3종**: 고객 / 공장 / 소핏 관리자
- **권한 분리 (RBAC)**:
  - 고객은 자기 견적만 조회·수정
  - 공장은 자기에게 매칭된 요청만 조회
  - 관리자는 전체 데이터 접근
- **이용약관·개인정보 처리방침 동의**: 회원가입 시 필수 체크
- **공장은 추가 심사 필요**: 가입 후 `승인 대기` 상태, 관리자 승인 후 `활성`

> ❓ **[미확정]**
> - 이용약관·개인정보 처리방침 초안 (Claude와 작성, 변호사 검수 필요)

### 4.3 [P0] 고객 화면

#### 4.3.1 견적 요청 폼 (10개 카테고리)

천일쇼파 견적서 기반으로 도출한 표준 폼 구조. **필수(\*)** / 선택 표기.

**① 고객 / 현장 정보**
- 고객사 **\*** / 연락처 **\*** / 현장명 **\***
- 업종 (카페 / 호텔 / 오피스 / 학원 등) — 선택
- 주소 **\*** / 층수 **\*** / 엘리베이터 여부 **\***
- 주차·상차 가능 여부 — 선택
- 현장 담당자 **\*** / 시공 가능 시간 **\***

**② 제품 / 쇼파 정보** (모두 선택)
- 쇼파 타입 — 일자형, ㄱ자, U자, 붙박이, 모듈형, 자유형
- 쇼파 개수 / 좌석 수 / 등받이 높이 / 팔걸이 여부
- 쿠션 구조 / 하부 프레임 구조 / 난연 여부 / 방수 여부

**③ 규격 (사이즈)** (모두 선택)
- 전체 길이, 폭, 높이 / 좌방석 높이, 깊이
- 벽면 길이, 코너 각도
- 도면 / CAD / PDF 업로드

**④ 자재** (모두 선택)
- 원단 — PU, 패브릭, 벨벳, 레자, 천연가죽, 기능성 원단
- 내부재 — 기본, 메모리폼, 하드폼
- 프레임 — 목재, 철제
- 컬러 / 샘플 이미지

**⑤ 공정 정보** — 현장 실측 여부

**⑥ 설치** — 설치 예상 시간

**⑦ 일정** — 실측일 / 제작 시작·완료일 / 출고일 / 설치일 / AS 가능일

**⑧ 비용** ⏸ *Phase 1+ (공장이 견적서 작성 시 입력)*

**⑨ 품질 / AS** ⏸ *Phase 1+ (사후 관리)*

> ✅ **[확정]**
> - 10개 카테고리 폼 구조 확정 (천일쇼파 견적서 기반)
> - **구현**: Next.js + React Hook Form + Zod
> - 단계별로 채울 수 있게 UI 설계 (다단계 폼)

> 📝 **[검토]**
> - 빌트인 가구 전용 폼 추가 항목 정리 → M+1 ~ M+2 보완

#### 4.3.2 내 견적 현황

- 내가 요청한 견적 목록 (상태별 필터)
- **견적 상태**: `요청됨` → `검토중` → `견적서 도착` → `소통중` → `계약 완료` → `시공중` → `완료`
- 각 견적 상세 페이지 (받은 견적서, 채팅, 진행 사진)

#### 4.3.3 받은 견적서 비교

- 1~3개 공장의 견적서 카드 형태로 비교
- 가격·납기는 고객에게만 공개 (공장 간 비공개)
- 공장 포트폴리오 링크 노출

#### 4.3.4 공장 포트폴리오 둘러보기

- 비로그인 사용자도 접근 가능 (전체 공개)
- 공장별 작업 사례 갤러리

### 4.4 [P0] 공장(제작업체) 화면

#### 4.4.1 가입·심사

- 사업자등록증 업로드 (Supabase Storage)
- 포트폴리오 등록 (최소 3개 작업 사례)
- 상태: `승인 대기` → 관리자 심사 → `활성` / `반려`

#### 4.4.2 포트폴리오 관리

- 작업 사례 등록·수정·삭제
- 사진 / 작업 설명 / 카테고리 (쇼파 / 빌트인) / 완성 연월
- **전체 공개** (비로그인 사용자도 볼 수 있음)

#### 4.4.3 매칭된 요청 확인 + 수락/거절

- 소핏이 배포한 견적 요청 목록
- 요청 상세 (고객 정보 일부 + 폼 내용 + 사진)
- **[수락] / [거절]** 버튼 — 공장이 자율적으로 참여 여부 결정
- 수락 시 견적서 작성 화면으로 이동

#### 4.4.4 수정 견적서 작성

- 수락 후 수정 견적서 작성 화면
- 입력 항목: 자재비 / 제작비 / 운송비 / 설치비 / 철거비 / 추가비용 / 마진 / 견적서 버전
- 작성 후 **고객에게 직접 전달** (소핏 검수 단계 없음)

#### 4.4.5 진행 프로젝트 관리

- 진행 중 프로젝트 목록 (상태별)
- 진행 사진 업로드 ⏸ *AS 보증금 인센티브 자동화는 Phase 0.5*
- 일정 관리

#### 4.4.6 고객과의 직접 채팅

- 매칭 확정 후 고객↔공장 직접 채팅방
- 일정·자재·납기 등 작업 전반 직접 협의
- 4.7 채팅 시스템 참조

### 4.5 [P0] 관리자(소핏 백오피스) 화면 — 가장 중요!

**1차 MVP에서 관리자 화면이 가장 중요한 이유**: AI가 없으므로 사람이 모든 흐름을 직접 운영해야 함.

#### 4.5.1 전체 견적 요청 목록

- 상태별 필터 (`요청됨` / `검토중` / `견적서 도착` / `완료` 등)
- 요청 상세 (고객 정보 / 폼 / 사진 / 진행 이력)

#### 4.5.2 공장 배포 UI ★

- 견적 요청을 검토 후 적합한 공장들에게 배포
- 여러 공장에게 동시 배포 가능 (공장이 수락/거절 자율 결정)
- 배포 이력 및 공장별 수락 현황 관리

#### 4.5.3 최종 매칭 확정 UI

- 고객이 선택한 공장을 확인하고 최종 매칭 상태 확정
- 매칭 확정 시 고객↔공장 직접 채팅방 자동 생성
- 미선택 공장에게 미채택 알림 발송

#### 4.5.4 공장 관리

- 가입 신청 공장 목록 (`승인 대기` 상태)
- 사업자등록증·포트폴리오 검토 후 승인/반려
- 활성 공장 목록 (활성 / 정지 처리)

#### 4.5.5 채팅 모니터링 뷰

- 매칭 성사 전 고객↔소핏 채팅 관리 (문의 응대)
- 매칭 성사 후 고객↔공장 직접 채팅 모니터링 (읽기 전용)
- 분쟁 발생 시 중재를 위해 채팅 기록 열람 가능

#### 4.5.6 사용자 관리

- 고객·공장 계정 목록
- 상태 변경, 계정 정지 등

---

### 4.6 [P0] 매칭 시스템

- 관리자가 견적 요청 검토 → 적합한 공장들에게 배포 (수동)
- 공장들이 요청 검토 후 수락/거절 자율 결정
- 수락한 공장들이 각자 수정 견적서 작성 → 고객에게 직접 전달
- 고객이 견적서 비교 후 원하는 공장 선택
- 고객 선택 확정 → 소핏 관리자가 최종 매칭 확정
- 가격·납기는 고객에게만 공개 (공장 간 비공개)

> ✅ **[확정]**
> - 배포는 수동 (자동 매칭 X)
> - 입찰 방식: 다수 공장 참여 → 고객이 선택
> - 1차 MVP에서는 천일쇼파 단독, M+3 이후 확장

---

### 4.7 [P0] 채팅 시스템

매칭 성사 전후로 채팅 채널이 나뉜다.

**주요 요구사항:**

- **매칭 전** — 고객 ↔ 소핏 채팅 (문의·안내용, Supabase Realtime)
- **매칭 확정 후** — 고객 ↔ 공장 **직접 채팅** (소핏 경유 없음, Supabase Realtime)
- 소핏 관리자는 고객↔공장 채팅을 모니터링 가능 (읽기 전용)
- **채팅 내 파일 첨부** (사진, 도면)
- 채팅 기록 보존 (분쟁 시 증거)

> ✅ **[확정]**
> - 주요 채팅: 웹페이지 자체 개발 (Supabase Realtime + Next.js)
> - 카카오톡 알림톡: Phase 1+에서 솔라피 도입

---

### 4.8 [P0] 파일 업로드 시스템

**왜 핵심인가?**: 견적의 절반 이상이 사진·도면 기반.

**주요 요구사항:**

- 매장 사진 (3~10장)
- 도면 / CAD / PDF
- 사업자등록증
- 포트폴리오 작업 사진
- 진행 사진
- 채팅 첨부 파일

**스토리지: Supabase Storage**

**파일별 권한:**

- 매장 사진: 고객·매칭된 공장·관리자만 조회
- 사업자등록증: 관리자만 조회
- 포트폴리오: 전체 공개
- 채팅 첨부: 채팅 참여자만 조회

---

### 4.9 [P0] 알림 시스템 (웹 내)

**왜 핵심인가?**: 알림 없으면 사람이 매번 새로고침해야 함.

**주요 요구사항:**

- 웹 내 벨 아이콘 알림 (새 알림 카운트)
- 알림 종류:
  - 새 견적 요청 (관리자 대상)
  - 새 매칭 (공장 대상)
  - 새 견적서 도착 (고객 대상)
  - 새 채팅 메시지 (전원)
  - 상태 변경 (해당 사용자)
- 알림 클릭 시 해당 페이지로 이동

> ⏸ **[Phase 1+]**
> - 이메일·카카오톡 외부 알림 (솔라피)

---

### 4.10 [P0] 상태 관리 시스템

각 견적이 어느 단계에 있는지 양측 모두가 볼 수 있어야 함.

**견적 상태(Status) 흐름:**

```
요청됨 → 검토중 → 공장배포중 → 견적서 도착 → 계약 완료 →
시공중 → 완료
```

> 매칭 확정(계약 완료) 이후부터 고객↔공장 직접 채팅 채널이 활성화됨

**공장 상태:**

```
승인 대기 → 활성 / 반려 → 정지
```

**구현:**

- DB의 `status` 컬럼으로 관리
- 상태별 색상·아이콘 표시
- 상태 변경 이력 추적 (audit log)
