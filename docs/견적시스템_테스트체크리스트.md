# 견적 시스템(다분야 확장) 테스트 체크리스트

> 대상: Phase 4(동적 폼 스튜디오) + Phase 5(상세 3화면 동적 렌더)
> 방법: 실제 브라우저에서 수동 테스트. 각 항목 **동작 → 기대결과** 확인 후 `[x]` 체크.
> 작성일: 2026-06-29

---

## 0. 사전 준비

- [ ] 011 마이그레이션이 Supabase에 적용돼 있다. (이미 완료)
- [ ] 개발 서버 실행: 터미널에서 `npm run dev` → `http://localhost:3000`
- [ ] **고객 계정으로 로그인**돼 있다. (공장/관리자 계정 아님)
- [ ] 테스트 진입 주소: **`/customer/request/new`** ← 신규 다분야 폼
  - 참고: 기존 `/customer/request`(쇼파 전용 폼)는 그대로 남아 있음. 헷갈리지 말 것.

> 💡 한 화면(분야/작성/요약 3분할)은 **데스크톱 넓은 화면**에서 가장 잘 보입니다. 모바일은 상단 탭으로 전환됩니다.

---

## 1. 진입 · 임시저장(draft) 생성

- [ ] `/customer/request/new` 접속 시 로그인돼 있으면 폼이 뜬다. (비로그인이면 로그인 페이지로 이동)
- [ ] 화면이 **좌(분야 선택) / 중(작성) / 우(요약·제출)** 3분할로 보인다.
- [ ] 처음엔 중앙에 "왼쪽에서 시공 분야를 선택하면…" 안내가 보인다.
- [ ] 페이지를 새로고침해도 같은 draft가 유지된다(같은 화면). → 고객당 draft 1건 재사용.

---

## 2. 좌측 — 분야 선택 사이드바

- [ ] 맨 위 **"공통 정보"** 항목이 고정으로 보인다.
- [ ] 분야가 **공정 흐름순 그룹**(철거·기초 / 골조·목공 / 마감 / … / 마무리)으로 묶여 보인다.
- [ ] 스키마가 준비된 **8개 분야**(목공·바닥·도배·도장·조명·주방·욕실·쇼파·가구)는 **선택 가능**하다.
- [ ] 나머지 분야는 **"준비 중"**으로 흐리게 표시되고 선택 안 된다.
- [ ] 분야 체크박스를 켜면 → 중앙에 해당 분야 섹션이 **즉시 추가**된다.
- [ ] 분야를 끄면 → 중앙 섹션이 **사라진다**.
- [ ] 분야 여러 개(예: **목공 + 도장 + 조명**)를 동시에 선택할 수 있다.
- [ ] 선택한 분야 옆에 진행률(**% 또는 ✓**)이 표시된다.
- [ ] 분야명을 클릭하면 중앙의 해당 섹션으로 **스크롤 이동**한다.

---

## 3. 중앙 — 공통 정보 입력

- [ ] "공통 정보" 섹션에 **현장 기본 / 현장 조건 / 일정·예산** 그룹이 보인다.
- [ ] 필수 항목(현장명·주소·담당자명·연락처·건물 유형·공사 성격·예산 범위)에 **빨간 \* 표시**가 있다.
- [ ] **주소**: 도로명 + 상세주소 두 칸으로 입력된다.
- [ ] **면적**: 입력칸 옆 **평 / ㎡ 토글**이 있고, 전환하면 숫자가 환산돼 보인다. (예: 10평 ↔ 33㎡)
- [ ] **건물 유형 / 공사 성격 / 예산 범위**: 드롭다운에서 선택된다.
- [ ] **실측 필요** 체크박스(토글)가 켜고 꺼진다.
- [ ] 입력값을 바꾸면 우측 요약의 "미입력 필수" 숫자가 줄어든다.

---

## 4. 중앙 — 분야별 입력 (동적 필드)

> 예: **목공** 분야로 확인.

- [ ] 분야마다 입력 항목이 **다르게** 나온다. (목공=작업항목/공간, 도장=도장부위/면적 …)
- [ ] **다중 선택(multiselect)** 항목(예: 목공 "작업 항목")은 **칩 버튼**으로 여러 개 선택된다.
- [ ] **단일 선택(select)** 항목은 드롭다운으로 선택된다.
- [ ] **숫자 + 단위** 항목(예: 수량 "개", 면적 "㎡")이 정상 입력된다.
- [ ] **"모름/추천 요청/현장확인/실측"** 안전장치가 붙은 항목(예: 목공 "자재 등급"=추천 요청)에서
  - [ ] 체크하면 입력 위젯이 **숨겨진다**.
  - [ ] 해제하면 다시 **입력 위젯이 나온다**.
- [ ] 도움말 텍스트(예: "잘 모르시면 추천 요청을 선택하세요.")가 보인다.
- [ ] **분야 제거** 버튼을 누르면 그 분야 섹션이 사라진다. (사이드바 체크도 해제됨)

### 쇼파 분야(기존 자산 이식) 확인
- [ ] 쇼파 분야 선택 시 형태/수량/등받이/쿠션/원단/치수 등 기존 항목이 모두 보인다.

---

## 5. 임시저장 · 복구

- [ ] 우측 **"임시저장"** 버튼을 누르면 상태가 "저장 중… → ✓ 저장됨"으로 바뀐다.
- [ ] 입력 도중 **페이지 새로고침** 후에도 작성 내용이 그대로 복구된다. (localStorage)
- [ ] 분야를 켜고 끈 뒤 새로고침해도 선택 상태가 유지된다.
- [ ] (선택) 브라우저를 닫았다 다시 `/customer/request/new` 들어가도 내용이 남아 있다.

---

## 6. 제출 — 검증 & 성공

### 검증 실패 케이스
- [ ] 필수 항목을 **비운 채** "견적 요청 제출"을 누르면 제출되지 않는다.
- [ ] 미입력 필수 필드가 **빨갛게 하이라이트**되고, 첫 항목으로 **자동 스크롤**된다.
- [ ] 분야를 하나도 선택 안 하고 제출하면 "최소 한 개 분야…" 안내가 뜬다.

### 제출 성공 케이스
- [ ] 공통 필수 + 분야 1개 이상(분야 필수 채움) 입력 후 제출하면
  - [ ] **"견적 요청이 접수됐습니다"** 완료 페이지로 이동한다.
  - [ ] (확인) 제출 후 localStorage 임시저장이 비워진다(새 draft로 다시 들어가면 빈 폼).

---

## 7. 상세 화면 동적 표시 (Phase 5)

> 방금 제출한 요청을 각 화면에서 확인.

- [ ] **고객** `/customer/requests` → 방금 요청 클릭 → 상세에 **"공통 정보" + "분야별 요청 내용"**이 분야별 카드로 보인다.
- [ ] 선택했던 분야(목공·도장·조명 등)가 **모두** 표시되고, 입력한 값이 라벨로 정확히 보인다. (예: select 값이 한글 라벨로)
- [ ] "모름/실측" 선택한 항목은 **노란 뱃지**로 표시된다.
- [ ] 주소가 도로명+상세로 표시된다.
- [ ] (관리자 계정 있으면) **관리자** `/admin/requests/{id}` 에서도 분야별로 동일하게 보인다.
- [ ] (관리자에게 매칭된 공장 계정 있으면) **공장** 상세에서도 분야별 내용이 보인다.
- [ ] 기존(레거시) 쇼파 요청을 열면 **기존 방식 그대로** 잘 보인다(깨지지 않음).

---

## 8. 보안 · 권한 (간단 확인)

- [ ] 로그아웃 상태로 `/customer/request/new` 접속 → 로그인 페이지로 막힌다.
- [ ] 다른 고객의 요청 상세 URL(`/customer/requests/{남의id}`)로 접근 → 안 보인다(404/차단).
- [ ] 공장/관리자 계정으로 `/customer/request/new` 접속 → 고객 화면이 아니라 자기 역할 페이지로 리다이렉트된다.

---

## 9. 반응형(선택)

- [ ] 모바일 폭으로 줄이면 상단에 **[분야] [작성] [요약]** 탭이 나타난다.
- [ ] 탭을 전환하면 각 영역이 단독으로 보인다.

---

## 알려진 제한 (테스트 대상 아님)

- ⏸ **사진·도면 첨부**: 신규 폼에는 아직 없음(Phase 8 예정). 첨부 없이 제출 가능.
- ⏸ **분야별 업체 매칭**: 제출 시 분야별 매칭은 아직 생성 안 됨(Phase 7). 현재는 상태 전환 + 관리자 알림 + 채팅방 생성까지.
- ⏸ **서버 자동저장(디바운스)**: 현재는 localStorage 즉시 + 분야토글·버튼·제출 시 서버 저장(Phase 8에서 자동저장 고도화).
