# 운영 후 할 일 (Post-Launch Backlog)

MVP·보안 하드닝은 완료. 아래는 **실서비스가 굴러가기 시작한 뒤** 또는 **외부 계정/데이터가 준비됐을 때** 착수하는 항목들.
작성: 2026-07-01. 각 항목은 "무엇을 / 왜 / 언제(트리거) / 어떻게" 순.

---

## A. 외부 서비스 연동 (계정만 준비되면 바로)

코드는 이미 붙어 있거나 소량 작업만 남음. **계정 가입 + 키/URL만** 있으면 됨.

- [ ] **Slack 실시간 보안 알림**
  - 무엇: 보안 이벤트 급증 시 Slack으로 즉시 통지.
  - 이미 됨: `logSecurityEvent`에 임계치 알림 로직 구현 완료(env-gated).
  - 어떻게: Slack 워크스페이스 → Incoming Webhooks에서 URL 발급 → `SECURITY_ALERT_WEBHOOK_URL` 환경변수에 설정 후 배포. 끝.
- [ ] **Sentry (에러 추적)**
  - 무엇: 프로덕션 런타임 에러·예외를 실시간 수집·알림.
  - 언제: 실사용자 트래픽 시작 직전/직후.
  - 어떻게: Sentry 프로젝트 생성 → DSN 발급 → Next.js SDK 연동(코드 작업 필요, DSN 주면 진행).
- [ ] **업타임 모니터**
  - 무엇: 사이트 다운·응답 지연 감지 및 알림.
  - 어떻게: UptimeRobot/BetterStack 등에서 헬스 URL 등록(가입만, 코드 거의 불필요).
- [ ] **비용 알림** (CLAUDE.md 과금 보안)
  - 무엇: Supabase·Vercel·(사용 시)OpenAI 등 월 비용 급증 감지.
  - 어떻게: 각 콘솔에서 budget/usage alert 설정. 외부 API 키에 usage limit 지정.

## B. 운영 데이터가 쌓인 뒤 (Phase 8 3차)

지금 만들어도 쓸 데이터가 없어 보류. **응답·낙찰·수정 이력이 축적된 뒤** 가치 발생.

- [ ] **자동 업체 추천 고도화**
  - 무엇: 현재는 "분야 태그 일치" 공장에 매칭 알림. 여기에 지역 근접·평점·응답속도·과거 낙찰률 기반 **추천 순위**를 얹음.
  - 언제: 공장별 응답/낙찰 이력이 유의미하게 쌓인 뒤(추천의 근거 데이터).
- [ ] **revisions 변경 이력 뷰**
  - 무엇: 제출 시점 스냅샷을 이미 `quote_request_revisions`에 저장 중. 이를 **관리자 상세에서 열람 + 버전 간 diff**로 노출(분쟁·정산 근거).
  - 언제: 요청 수정이 실제로 오가기 시작하면. ※ 저장은 이미 되고 있어 **뷰만 붙이면 되는 예외 항목** — 데이터 없이도 착수 가능.
- [ ] **스키마 DB 이전**
  - 무엇: 27개 분야 항목 정의가 코드(`fieldSchemas.ts`)에 하드코딩됨. 이를 DB로 옮겨 **배포 없이 관리자가 항목 편집** 가능하게.
  - 언제: 분야 항목을 자주 손봐야 할 필요가 생기면.

## C. 검증 (실계정/브라우저 준비되면)

- [ ] **인증 사용자 간 IDOR 라이브 테스트**
  - 무엇: A공장 계정으로 B공장의 견적/매칭 조회, 고객A가 고객B 요청 접근 등 **로그인 상태의 권한 경계**를 실제 조작으로 확인.
  - 이미 됨: 미인증(anon) 경계는 `scripts/verify-rls.mjs`로 실측(17/17 PASS). 코드 소유권 검증도 감사 완료.
  - 어떻게: 고객·공장·관리자 테스트 계정 각 1개 준비 → 크로스 접근 시도.
- [ ] **전체 화면 흐름 수동 검증**
  - 요청 작성 → 제출 → 매칭 → 분야별 견적 → 분야별 수락 → 채팅까지 실제 클릭으로 1회 관통.

## D. 정기 운영 보안 (주기적 반복)

CLAUDE.md 보안 기준 중 "1회성"이 아니라 **운영하며 계속** 챙길 것.

- [ ] **키 로테이션** — service_role·API 키 최소 90일 주기 교체.
- [ ] **개인정보 보관/파기** — `security_events`·로그 등 90일 경과분 정기 삭제(예: `DELETE FROM security_events WHERE created_at < now() - interval '90 days'`). 보유기간 지난 개인정보 파기.
- [ ] **의존성 취약점 스캔** — `npm audit` 정기 실행, 알려진 CVE 패치.
- [ ] **보안 회귀 점검** — 배포 전 `scripts/verify-rls.mjs` 재실행으로 RLS 경계 유지 확인.
- [ ] **레거시 데이터 정리** — 비공개 전환 전 저장된 chat-attachments 공개 URL(있다면) 및 미사용 첨부 정리.
